# 未实现功能清单0

本文档只记录当前仍未真正实现、仍是占位或仅保存配置但没有业务闭环的功能。已完成并通过 GUI/自检验证的局域网发现、真实 UDP 发送、接收确认、失败重试、结果统计、演示设备兜底移除和 `acco` 账号表读写等能力不再重复列入。

## 传输内核

### 下载限速未形成业务闭环

当前状态：系统设置页可以填写并保存 `downloadLimit`，但代码中只有发送端 `UdpTx` 使用 `uploadLimit` 做上传限速分配，接收端没有按 `downloadLimit` 做收包节流或背压。

未实现原因：UDP 接收侧限速不能只靠简单 sleep，否则可能导致系统 UDP 缓冲区溢出，需要结合发送端 ACK 节奏、接收缓存和丢包重传策略一起设计。

推荐补法：优先把下载限速解释为“接收端 ACK 放行速率”，在 `UdpRx` 对 DATA 包验收后按设置延迟 ACK，让发送端自然受控；同时在说明书中写清楚这是接收侧限速而不是网络驱动层限速。

### 文件类型匹配规则未实现

当前状态：文件拖拽、文件选择和文件夹选择可以正常加入待传输列表，`FileIcons` 只按扩展名/文件夹内容显示图标；目前没有按接收对象、文件类型白名单或业务分类做“类型匹配/拒收”。

未实现原因：实验报告提到“拖拽文件类型匹配”，但现有前端没有给出明确的匹配规则，例如按图片/文档/压缩包筛选、按用户偏好筛选，还是只做图标识别。

推荐补法：先在设置或用户资料中增加可接收类型偏好，再在发送前和接收 BEGIN 阶段同时校验扩展名/MIME 兜底；规则明确前保持“全部类型可传输”比误拦截更适合当前演示。

## 传输协作能力

### 辅助账号 push token 未配置到运行环境

当前状态：App 已支持生成 `req/<账号>` 并直接 `git push` 到仓库，再由 GitHub Actions 合入 `acco`。普通用户机器可以通过运行时 `ACCO_T` 环境变量或 `-Dacco.t=...` JVM 参数注入辅助账号 token；如果没有 token，注册 push 会明确失败，不退回用户本机主 GitHub 账号凭据。

未实现原因：真实 token 属于密钥，不能写进仓库、说明书、代码或提交记录；GitHub 也不支持用邮箱密码直接做 Git/API 认证。当前代码闭环已完成，但分发或演示机器仍需要在运行环境外部配置 token。

推荐补法：使用隔离浏览器登录辅助账号，在 GitHub 创建只授予本仓库 `Contents: Read and write` 的 fine-grained token；运行 App 前把 token 放入 `ACCO_T` 或启动参数 `-Dacco.t=...`。若后续打包分发，建议用外部启动脚本或用户本机环境变量配置，不要把 token 放入项目文件。

### 系统托盘与完成提示音未实现

当前状态：设置页可以保存 `startMinimized` 和 `soundOnComplete`，但代码只落盘，没有真正创建系统托盘图标、启动后最小化到托盘，也没有在传输完成后播放提示音。`autoStart` 已由 `AutoStart` 写启动脚本，不属于本项。

未实现原因：JavaFX 跨平台托盘和音频播放需要额外处理 AWT/SystemTray、资源文件、窗口生命周期和权限问题，和当前后端传输主线关系较弱。

推荐补法：Windows 课程演示可先用 AWT `SystemTray` 和 JDK `Toolkit.beep()` 做轻量闭环；如果要求更完整，再补托盘菜单、恢复窗口和可配置提示音资源。

### 多语言界面未实现

当前状态：语言设置可以保存 `language=简体中文`，资料也保留语言字段，但界面文案仍是硬编码中文，没有资源包或运行时文案切换。

未实现原因：完整多语言需要把所有页面文案抽离为资源表，并处理控件宽度、排序、日期格式和测试截图，不适合夹在传输后端主线中零散完成。

推荐补法：若后续必须实现，新增 `i18n/messages_zh.properties` 和 `messages_en.properties`，先覆盖导航、按钮、表头、toast 和弹窗，再逐页替换硬编码文案。
