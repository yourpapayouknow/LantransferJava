# 未实现功能清单0

本文档只记录当前仍未真正实现、仍是占位、演示兜底或仅保存配置但没有业务闭环的功能。

## 传输内核

1. 真实 UDP 多线程传输调度与实时进度
   - 当前状态：部分未实现。`UdpTx` 和 `UdpRx` 已实现目标级并发 UDP 文件发送、停止等待 ACK、后台接收和接收目录落盘。
   - 缺口：还没有传输中实时进度推送、接收端进度展示、队列调度和分片级并发。
   - 推荐实现：在现有 `UdpTx/UdpRx` 协议上增加任务状态仓库，把已确认分片数暴露给 UI；只有在大文件性能不足时再把单文件分片分派到并发 worker。

2. 文件临时分片缓存与断点重传
   - 当前状态：未实现持久断点重传。`UdpRx` 已使用 `.part` 临时文件和内存 `BitSet` 记录当次接收分片。
   - 缺口：分片状态没有持久化到磁盘，应用重启后不能继续；发送端也没有按接收端缺失分片列表定向补发。
   - 推荐实现：为每个任务保存 chunk bitmap 和元数据文件，重启后恢复 `.part` 状态，并提供缺失分片查询和补发协议。

3. 传输预计剩余时间统计
   - 当前状态：`UdpTx` 只在发送完成后计算总耗时和平均速度。
   - 缺口：没有真实实时进度、速率采样和剩余字节统计。
   - 推荐实现：在真实传输任务中按已确认分片数和滑动平均速度计算 ETA。

## 传输协作能力

1. 基于传输口令的虚拟小群组传输
   - 当前状态：未实现。
   - 缺口：UI 没有口令输入，发现协议没有群组字段。
   - 推荐实现：先在设置页或发送页增加口令字段，再把口令哈希加入发现和发送握手协议。

2. 基于用户设定状态的条件文件传输
   - 当前状态：只保存了用户状态和自定义签名。
   - 缺口：发现协议尚未广播状态，发送器也没有按 ONLINE/BUSY/INVISIBLE/OFFLINE 做拦截。
   - 推荐实现：扩展 `LanPeer` 响应字段携带 `UserStatus`，发送前按状态执行接收策略。

3. 忙碌状态下的接收前确认
   - 当前状态：未实现。
   - 缺口：没有接收端确认弹窗和发送端等待确认流程。
   - 推荐实现：真实接收服务收到请求后先进入待确认状态，用户确认后再返回 ACK。

## 系统设置

1. 系统级开机自启动
   - 当前状态：开机自启动、启动后最小化、传输完成提示音已经保存到 `SystemSettings`。
   - 缺口：开机自启动还没有写入 Windows 注册表或启动目录快捷方式。
   - 推荐实现：在 Windows 下创建启动目录快捷方式，关闭时删除。

## 账号体验

1. 注册审核流程
   - 当前状态：本地注册立即成功并返回登录页；审核等待页仅保留给未来服务端审核。
   - 缺口：没有管理员、服务器或审核队列。
   - 推荐实现：如果需要课堂展示审核，使用本地 pending 注册文件；真实多人审核需要服务端。
