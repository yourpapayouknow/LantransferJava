# 未实现功能清单0

本文档只记录当前仍未真正实现、仍是占位、演示兜底或仅保存配置但没有业务闭环的功能。

## 传输内核

1. 真实 UDP 多线程文件发送与接收落盘
   - 当前状态：未实现。现在由 `TxSim` 根据真实文件和目标生成传输报告。
   - 已推进：`UserDevice` 已有目标 IP/端口，`LanPeer` 已广播传输地址，系统设置已保存接收目录。
   - 缺口：还没有后台接收服务、接收确认、接收进度和真实发送器。
   - 推荐实现：新增后台 `UdpRx` 接收服务，再实现 `UdpTx` 分片发送、ACK 和多线程调度。

2. 文件临时分片缓存与断点重传
   - 当前状态：未实现。现在只有 `TxSim` 的失败重试统计。
   - 缺口：缺少真实 UDP 分片协议、接收端缓存目录、传输任务 ID 和文件写入状态。
   - 推荐实现：真实 `UdpTx/UdpRx` 完成后，为每个文件生成任务 ID 和 chunk index，保存 `.part` 缓存和 bitmap 进度。

3. 文件接收完整性校验
   - 当前状态：未实现。
   - 缺口：没有真实接收端落盘流程，无法对完整文件做校验。
   - 推荐实现：接收完成后用 SHA-256 校验整文件，失败时触发缺失分片重传。

4. 根据限速智能分配带宽
   - 当前状态：只保存了上传/下载限速配置，没有真实传输调度。
   - 缺口：没有真实发送流，无法执行速率控制。
   - 推荐实现：在 UDP 发送器中用令牌桶按目标数分配发送速率。

5. 传输预计剩余时间统计
   - 当前状态：`TxSim` 只按固定模拟速度生成总耗时，不是真实剩余时间。
   - 缺口：没有真实实时进度、速率采样和剩余字节统计。
   - 推荐实现：在真实传输任务中按已确认分片数和滑动平均速度计算 ETA。

## 传输协作能力

1. 基于传输口令的虚拟小群组传输
   - 当前状态：未实现。
   - 缺口：UI 没有口令输入，发现协议没有群组字段。
   - 推荐实现：先在设置页或发送页增加口令字段，再把口令哈希加入发现和发送握手协议。

2. 基于用户设定状态的条件文件传输
   - 当前状态：只保存了用户状态和自定义签名。
   - 缺口：发现协议尚未广播状态，发送器也没有按 ONLINE/BUSY/INVISIBLE/OFFLINE 做拦截。
   - 推荐实现：扩展 `LanPeer` 响应字段携带 `UserStatus`，发送前按状态执行接收策略。

3. 忙碌状态下的接收前确认
   - 当前状态：未实现。
   - 缺口：没有接收端确认弹窗和发送端等待确认流程。
   - 推荐实现：真实接收服务收到请求后先进入待确认状态，用户确认后再返回 ACK。

## 用户与设备列表

1. 数据库记录近期传输对象
   - 当前状态：未实现数据库或本地持久化。近期对象主要保存在 `MainWindow.recentTargets` 内存中，空网段时仍使用 `MockBackendFacade` 演示数据。
   - 缺口：重启后近期对象不会按真实传输历史恢复。
   - 推荐实现：用本地 properties 或简单 JSON 文件保存最近目标 ID、昵称、设备名和最后传输时间。

2. 用户列表搜索
   - 当前状态：搜索框已显示，但没有筛选逻辑。
   - 缺口：输入昵称或设备 ID 不会改变列表。
   - 推荐实现：给搜索框监听文本变化，对 `loadAllDevices()` 结果按昵称、设备名、ID 过滤。

3. 真实在线检测
   - 当前状态：UDP 扫描可发现同程序设备，但用户列表在只发现本机时回退到演示列表；离线状态多为演示数据。
   - 缺口：没有心跳、过期时间和离线判定。
   - 推荐实现：`LanPeer` 增加 lastSeen 时间戳，超过阈值标记 OFFLINE。

## 页面按钮与操作

1. 清除已完成传输记录
   - 当前状态：按钮仍提示“接口已预留”。
   - 缺口：没有传输历史仓库。
   - 推荐实现：先落地传输历史，再删除状态为“已完成”的记录。

2. 清空传输日志
   - 当前状态：按钮仍提示“接口已预留”。
   - 缺口：日志只存在当前 `TransferSummary` 内存对象中，没有清空接口。
   - 推荐实现：清空当前汇总日志并刷新结果页；若后续有日志文件，同步清理文件记录。

3. 更改接收文件存储位置
   - 当前状态：接收目录字段和设置页保存已实现，底部状态栏更改按钮会跳转设置页。
   - 缺口：接收端尚未落地，所以目录还没有被真实接收服务使用。
   - 推荐实现：实现 `UdpRx` 后写入 `SystemSettings.receiveDir()`。

4. 资料编辑按钮
   - 当前状态：“昵称/设备名称/个性签名”的编辑按钮只展示，没有进入编辑态。
   - 缺口：资料页目前只能保存现有 `Profile` 和自定义状态文本。
   - 推荐实现：把对应行切换为输入框，保存后调用 `updateProfile(...)`。

5. 状态卡点击切换
   - 当前状态：状态卡只展示，真正可保存的是自定义状态输入框。
   - 缺口：点击“在线/忙碌/隐身/离线”不会更新 `UserStatus`。
   - 推荐实现：给状态卡绑定点击事件并调用 `updateStatus(...)`。

## 系统设置

1. 系统级开机自启动
   - 当前状态：开机自启动、启动后最小化、传输完成提示音已经保存到 `SystemSettings`。
   - 缺口：开机自启动还没有写入 Windows 注册表或启动目录快捷方式。
   - 推荐实现：在 Windows 下创建启动目录快捷方式，关闭时删除。

2. 字体和缩放实际应用
   - 当前状态：字体、字号、缩放可以保存，但没有全局应用到 UI。
   - 缺口：CSS 和窗口缩放没有读取这些字段。
   - 推荐实现：在 `MainWindow` 应用根节点上写入字体和缩放样式变量。

## 账号体验

1. “记住我”
   - 当前状态：登录请求里有 `rememberMe` 字段，但后端没有保存最近登录账号或令牌。
   - 缺口：重启后不会自动填充或免登录。
   - 推荐实现：本地保存最近账号；如要免登录，再增加短期 token 文件。

2. 注册审核流程
   - 当前状态：本地注册立即成功并返回登录页；审核等待页仅保留给未来服务端审核。
   - 缺口：没有管理员、服务器或审核队列。
   - 推荐实现：如果需要课堂展示审核，使用本地 pending 注册文件；真实多人审核需要服务端。
